Đệ quy (Recursion)

Hàm tự gọi chính mình 🌀

🌀 Đệ quy là gì?

🪞
Hàm gọi lại chính nó với bài toán nhỏ hơn
Giống như gương soi gương — nhưng phải có điểm dừng!
def giai_thua(n):
    if n <= 1:        # Base case — điểm dừng!
        return 1
    return n * giai_thua(n - 1)  # Gọi lại chính mình

print(giai_thua(5))  # 5*4*3*2*1 = 120

📐 2 thành phần bắt buộc

1️⃣Base case — Điều kiện dừng (không gọi đệ quy nữa)
2️⃣Recursive case — Gọi lại với bài toán nhỏ hơn
⚠️ Quên base case → Stack Overflow (chạy mãi không dừng!)

🐇 Fibonacci

F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2)

# Cách 1: Đệ quy thuần (chậm!)
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)

# Cách 2: Đệ quy có nhớ (nhanh!)
memo = {}
def fib_memo(n):
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib_memo(n-1) + fib_memo(n-2)
    return memo[n]
💡 Đệ quy thuần: O(2ⁿ) — rất chậm! Có nhớ: O(n) — siêu nhanh!

🏆 Bài CP: Tính lũy thừa nhanh

def power(a, n):
    if n == 0: return 1
    if n % 2 == 0:
        half = power(a, n // 2)
        return half * half
    return a * power(a, n - 1)

# power(2, 10) = 1024
# Chỉ cần ~log₂(n) phép nhân thay vì n phép!
✅ Từ O(n) → O(log n) — kỹ thuật quan trọng trong CP!

🏗️ Tháp Hà Nội

def hanoi(n, src, dst, aux):
    if n == 1:
        print(f"Di chuyển đĩa 1: {src} → {dst}")
        return
    hanoi(n-1, src, aux, dst)
    print(f"Di chuyển đĩa {n}: {src} → {dst}")
    hanoi(n-1, aux, dst, src)

hanoi(3, 'A', 'C', 'B')

💪 Bài tập

Bài 1: Tổng đệ quy ⭐
Viết hàm sum_rec(n) = 1+2+...+n bằng đệ quy
Bài 2: Fibonacci thứ N ⭐⭐
Tính F(n) với memo, n ≤ 10⁶
Bài 3: Lũy thừa nhanh ⭐⭐⭐ CP
Tính a^n mod m (n ≤ 10¹⁸)
Bài 4: GCD đệ quy ⭐⭐ CP
Viết GCD bằng đệ quy: gcd(a,b) = gcd(b, a%b)
